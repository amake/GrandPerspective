Saving scan results

There are two ways in which you can save the contents of views, both accessible from the "File" menu:

  • The Save Scan Data command lets you save all data in the view. Data is saved to an XML file. It does not matter which folder is currently displayed in the view, all files —also those not currently shown in the view— are stored. However, if a filter has been used to create the view then files that have been excluded by the filter will not be stored.

    The saved data can be loaded into a new view at a later moment using the Load Scan Data command. Saving and loading data be useful for the following reasons:

    • You can view and navigate old contents of folders, even when these contents have now changed. You can even compare the old contents with the current contents to see where all your disk space has gone. Note: If you have Time Machine, you can also view the old contents of folders by scanning folders containing old Time Machine back-ups,
    • You can view and navigate scanned folder contents at another machine.
    • Loading saved scan data is quicker than obtaining the same data by scanning the folder again.
    • Experienced users may process, analyse, and manipulate the saved data using shell scripts or other tools.

    Beware: When viewing loaded scan data, you have to be extra careful using the Delete command. Files or folders that you select for deletion may have different contents than the view suggests; folders may contain more files and files may have changed (in size) as well.

  • The Save Image of View command can be used to save a PNG image of the view. The image will be exactly the same as the image that is shown in the view. The size, colors and displayed files will all match the current view settings. You can use these images in many ways: frame them and hang them on your wall, upload them to your blog, email them to friends, etc.
