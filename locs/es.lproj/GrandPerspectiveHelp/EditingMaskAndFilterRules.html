Editando reglas de máscaras y filtrado

Cuando editas una máscara o filtro, puedes crear nuevas reglas o modificar las existentes. Puedes hacerlo desde la "Ventana editar reglas de filtros". Para las reglas que crees desde cero, puedes especificar un nombre para ayudar a distinguirla de las otras reglas.

Puedes escoger si la regla se debe aplicar a los archivos, carpetas, o ambos. Cuando la regla es aplicada a las carpetas, no podrás añadir las comprobaciónes a la regla que puedan ser aplicados solo a archivos.

Una regla debe consistir de una o más comprobaciones, que se habilitan con las cajas de selección (checkboxes) a la izquierda y pueden se configuradas entonces. Cuando la regla consiste de varias comprobaciones, todas deben mantenerse para coincidir con una regla para archivo o carpeta. Puedes escoger entre las siguientes comprobaciones:

  • Nombre: Comprueba si el nombre de un archivo o carpeta concuerda con algún objetivo. Por ejemplo, el nombre solo necesita coincidir con un objetivo para que la comprobación sea válida.
  • Ruta: Comprueba si la ruta de un archivo o carpeta coincide con alguno de los objetivos. Las rutas son absolutas. Comienza con la raíz del sistema e incluye el nombre del elemento. Para las carpetas, la ruta siempre termina con un separador de ruta, por ejemplo una barra ("/").
  • Tipo: Comprueba si el tipo de archivo es igual o se ajusta a cualquiera de los objetivos.
  • Hard-linked: Comprueba si el archivo o la carpeta son guardados en el sistema de archivos como hard link (enlace duro).
  • Paquete: Comprueba si el archivo o carpeta es un paquete. Dependiendo de si el contenido del paquete se muestra o no, el paquete es tratado como un archivo o como una carpeta.
  • Tamaño: Comprueba si el tamaño del archivo es superior o inferior al límite dado.
