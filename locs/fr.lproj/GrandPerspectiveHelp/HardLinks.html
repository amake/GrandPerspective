Lien matériel

Les Systèmes d'Exploitation basés sur Unix, incluant Mac OS X supportent le concept des liens matériels. Un lien matériel peut être utilisé pour placer un fichier à différents endroits (i.e. dans différents dossiers) ou sous différents noms dans le même système de fichiers. Chacun de ces liens matériels réfère aux mêmes données dans le système de fichiers ce qui ne prend de la place qu'une seule fois. Cela implique par exemple une modification de tous les liens matériels lorsque vous modifiez un fichier qui en est l'origine.

Les liens matériels doivent être gérés de manière spéciale afin de ne pas surestimer l'espace du disque dur qu'ils occupent. GrandPerspective gère les liens matériels comme suit. Quand un lien matériel (fichier ou dossier) est rencontré lors d'une analyse, il est inclus. Ultérieurement, si le même fichier ou dossier est rencontré une fois de plus (dans un dossier différent) il est ignoré. De ce fait, à l'intérieur d'une seule vue, un lien matériel apparaît au plus une fois, ce qui reflète correctement la taille qui est actuellement prise sur le disque. Toutefois, la question de la localisation d'un lien matériel apparait dans la vue est assez arbitraire. Il peut être dans n'importe quel dossier où il apparaît. Cela dépend du premier dossier qui aura été analysé.

Quand vous supprimez un fichier ou dossier qui est un lien matériel, il est supprimé du dossier donné. Toutefois, l'espace libre n'augmentera pas car l'objet prend de l'espace sur le disque tant que tous les liens matériels pointant vers ce dernier n'ont pas été supprimés.
