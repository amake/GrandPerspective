Comment analyser les sauvegardes de Time Machine

GrandPerspective peut être utilisé pour analyser les sauvegardes faites par Time Machine. Cela peut vous aider à optimiser vos méthodes de sauvegardes Time Machine, par exemple, en identifiant les fichiers et les dossiers qui ne devraient pas être sauvegardés.

Premièrement, il est très important de comprendre comment sont stockées les sauvegardes Time Machine. Time Machine fait un usage intensif des liens matériels. Chaque sauvegarde semble stocker une copie entière du disque qui est sauvegardé. Pourtant la quantité d'espace disque nécéssaire à une sauvegarde est limitée par les fichiers qui ont changé depuis la dernière sauvegarde. Tous les fichiers et dossiers qui n'ont pas changé sont representés par un lien matériel vers ces mêmes objets dans la précédente sauvegarde.

Ci-dessous, quelques exemples d'utilisation de GrandPerspective pour analyser vos sauvegardes Time Machine. Ces exemples sont juste là pour vous donner une idée de tout ce que vous pouvez faire. Vous êtes bien sûr libres d'expérimenter avec d'autres approches.

Comparaisons de sauvegardes

Vous souhaitez comparer deux sauvegardes pour voir ce qui a changé sur votre disque dur. C'est très simple. Premièrement, vous exécutez Analyse dossier... sur une ancienne sauvegarde, e.g. "Backup Volume/Backups.backupdb/My Computer/2008-08-08-888888". Ensuite, vous pouvez analyser le contenu de la dernière sauvegarde, i.e. "Backup Volume/Backups.backupdb/My Computer/Latest". En ayant les deux vues ouvertes l'une à côté de l'autre, vous devriez pouvoir repérer rapidement les grandes différences.

Note: Au lieu d'analyser la dernière sauvegarde, vous pouvez analyser le contenu actuel de votre disque dur. L'inconvénient de cette approche est que vos résultats contiendront les fichiers qui sont exclus de vos sauvegardes Time Machine, ce qui vous rendra la tâche plus difficile pour trouver des changements éventuels.

Analyser toutes les sauvegardes

Executez Analyse dossier... sur le dossier "Backup Volume/Backups.backupdb/My Computer". Cela peut prendre du temps. Quand l'analyse est terminée, une vue apparaît avec toutes les sauvegardes. Dans l'onglet Affichage, choisissez la coloration par "Top dossier". Tous les fichiers appartenant à la même sauvegarde auront alors la même couleur.

Vous allez sûrement voir qu'une sauvegarde est plus importante que les autres. C'est la sauvegarde qui a été analysée en premier (ce n'est pas nécéssairement la sauvegarde la plus récente). Tous les fichiers dans ce dossier n'avaient pas encore été rencontrés durant l'analyse et ont de ce fait été inclus. Quand l'analyse des autres dossiers de sauvegarde a été faites, la plupart des liens matériels auront déjà été rencontrés dans le premier dossier qui a été analysé et sont de ce fait, exclus.

Le dossier qui a été analysé en premier est actuellement le moins intéréssant. Il montre simplement le contenu de votre disque dur au moment de l'analyse (moins les fichiers que vous avez choisi d'exclure de vos sauvegardes Time Machine). Excluons le alors de notre vue. Commencez par éditer les masques de la vue. Vous devez créer une nouvelle règle. Nommez la "Sauvegarde principale", appliquez là à "fichiers et dossers" et rajoutez-y le chemin à tester, les chemins qui nous intéréssent sont ceux qui commencent par "/Volumes/Backup Volume/Backups.backupdb/My Computer/2008-08-08-888888". La partie en italique sera différente selon votre cas. Vous pouvez avoir les chemins dont vous avez besoin depuis l'onglet Focus. D'abord, dans la vue choisissez n'importe quel fichier à partir du plus grand dossier de sauvegarde, zoomé en avant d'un niveau, copier le chemin depuis la boîte de texte "Dossier en vue" et zoomé en arrière.

Après avoir créé la règle "Sauvegarde principale", marquez comme suit : "Ne pas montrer: Sauvegarde principale". Vous devriez alors voir que la sauvegarde principale est désormais masquée. Si ce n'est pas le cas, vous n'avez pas spécifié le masque correctement. Editez le encore jusqu'à ce que cela marche.

Quand le masque marche correctement, vous pouvez filtrer la vue. Utilisez la commande Filtre. Il devrait être configuré par défaut comme le masque, donc cliquez simplement sur "OK". Une nouvelle fenêtre est créée, contenant seulement les autres sauvegardes. Cette nouvelle vue donne une bien meilleure idée de la taille requise par chaque sauvegarde incrémentale. Naviguer et explorer cette vue pour avoir une meilleure idée de ce qui est sauvegardé. Si vous repérez des fichiers qui ne devraient pas être sauvegardés, mettez à jour vos préférences Time Machine.

Contrôler quels fichiers ont été sauvegardés durant la dernière sauvegarde

Executez Filtrez l'analyse.... Le dossier à scanner sera "Backup Volume/Backups.backupdb/My Computer/Latest". Appliquez les filtres suivants : "Ne pas montrer: les objets liens matériels". Le résultat de cette vue montre les fichiers qui sont nouveaux dans la dernière sauvegarde. Cette vue exclut tous les fichiers qui apparaissent dans la dernière sauvegarde.

Note: Vous pouvez appliquez le même filtre à d'autres dossiers de sauvegardes. Toutefois, vous devez alors interpréter les données de la vue différement. Dans ce cas, chaque fichier qui est visible n'apparaît pas dans la dernière sauvegarde et le contenu du dossier qui contient le fichier n'a pas changé dans les sauvegardes suivantes aussi longtemps que le fichier a existé (sans changement). En effet, c'est un peu plus dur à comprendre et légèrement moins utile en tant que résultat.
